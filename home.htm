Timestamp Microservice

http://pompan129-apiproj.herokuapp.com/timestamp/{DATE}
  • User Story: I can pass a string as a parameter, and it will check to see whether that string contains either a unix timestamp or a natural language date (example: January 1, 2016).
  • User Story: If it does, it returns both the Unix timestamp and the natural language form of that date.
  • User Story: If it does not contain a date or Unix timestamp, it returns null for those properties.

Example usage:

http://pompan129-apiproj.herokuapp.com/timestamp/July%2020,%201969
http://pompan129-apiproj.herokuapp.com/timestamp/-14256000000

Example Output:

{"unix":-14256000000,"natural":"JULY 20, 1969"}

URL Shortener Microservice

http://pompan129-apiproj.herokuapp.com/new/{URL}
  • User Story: I can pass a URL as a parameter and I will receive a shortened URL in the JSON response.
  • User Story: If I pass an invalid URL that doesn't follow the valid http://www.example.com format, the JSON response will contain an error instead.
  • User Story: When I visit that shortened URL, it will redirect me to my original link.

** short-urls are actually longer because of the lengthy Heroku domain. But you get the idea

Example URL Creation:

http://pompan129-apiproj.herokuapp.com/new/www.netflix.com

Example Creation Output:

{"orig_url":"http://www.netflix.com","short_url":"https://pompan129-apiproj.herokuapp.com/new/8"}

Example Usage:

https://pompan129-apiproj.herokuapp.com/new/8

Will redirect to:

http://www.netflix.com

Image Search Abstraction Layer

http://pompan129-apiproj.herokuapp.com/imagesearch/{Search String}
  • User Story: I can get the image URLs, alt text and page urls for a set of images relating to a given search string.
  • User Story: I can paginate through the responses by adding a ?offset=2 parameter to the URL.
  • User Story: I can get a list of the most recently submitted search strings.

Example Image Search:

http://pompan129-apiproj.herokuapp.com/imagesearch/spiderman?offset=5
